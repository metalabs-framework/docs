# MetaLabs Framework

A standalone, server-authoritative FiveM framework built from the ground up.

## Resources

| Resource | Description |
|----------|-------------|
| [meta_core](/meta_core) | Server-only authoritative core — player sessions, identity, permissions, validation, and persistent state |
| [meta_logic](/meta_logic) | Client/server bridge — network transport, reactive cache, player tracking, vehicle utilities, and interaction layer |
| [meta_lib](/meta_lib) | Shared utility library — callbacks, notifications, progress bars, validation, formatting, locale, cooldowns, caching, and result patterns |
| [meta_eco](/meta_eco) | Server-only economy authority — accounts, transactions, invoices, payroll, taxes, loans, credit scores, and business finance |

## Philosophy

- **Server-authoritative** — the server owns all truth; clients receive push updates
- **Framework-agnostic** — standalone core with a separate bridge layer for qb/qbx/esx compatibility
- **Modular** — each concern is isolated into its own module with clean boundaries
- **Minimal** — no bloat, no unnecessary abstractions, just what's needed

## Tech Stack

- Lua 5.4 on FiveM (cerulean)
- OneSync
- oxmysql
