# State

The state module provides per-player key/value storage that persists to MySQL. State is saved as a JSON blob in the `meta_player_state` table.

## How It Works

- State is loaded from the database when a player connects
- Changes are tracked with a dirty flag
- A background thread persists dirty state every `StateSaveInterval` seconds (default 300)
- State is also persisted when a player disconnects or the resource stops
- State changes fire the `stateChanged` event

## Exports

### PlayerGetState

Get a single state value by key, or the full state table.

```lua
local isDead = exports.meta_core:PlayerGetState(source, 'isDead')
local allState = exports.meta_core:PlayerGetState(source)
```

### PlayerSetState

Set a state value. Fires the `stateChanged` event with old and new values.

```lua
exports.meta_core:PlayerSetState(source, 'isDead', true)
exports.meta_core:PlayerSetState(source, 'money', 5000)
```

### PlayerSetStates

Set multiple state values at once. Fires the `stateBatchChanged` event.

```lua
exports.meta_core:PlayerSetStates(source, {
    isDead = false,
    armor = 100,
})
```

### PlayerRemoveState

Remove a state key.

```lua
exports.meta_core:PlayerRemoveState(source, 'isDead')
```

## Using meta.player directly

If your resource runs inside meta_core (or has access to the global):

```lua
meta.player.getState(source, 'isDead')
meta.player.setState(source, 'isDead', true)
meta.player.setStates(source, { isDead = false, armor = 100 })
meta.player.removeState(source, 'isDead')
```

## Storage

State is stored in the `meta_player_state` table:

| Column | Type | Description |
|--------|------|-------------|
| `identifier` | VARCHAR(60) | Player's unique identifier (primary key) |
| `state_data` | LONGTEXT | JSON-encoded state bag |
| `updated_at` | TIMESTAMP | Last persistence time |
