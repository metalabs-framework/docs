# Network

The network module provides a single request/response channel between client and server. Instead of registering individual net events per feature, all communication flows through a route-based dispatcher.

## How It Works

1. Client sends a request with a route name and payload
2. Server dispatcher finds the registered handler for that route
3. Handler processes the request and returns data (or error)
4. Response is sent back to the client and resolves the pending request

Only **two** net events are used for all communication: `NET_REQUEST` and `NET_RESPONSE`.

## Client Usage

### Callback Style

```lua
ml.Network.Request('player:getCharacter', {}, function(data, err)
    if err then
        print('Error:', err)
        return
    end
    print(data.fullname)
end)
```

### Async Style (Recommended)

Uses `promise` + `Citizen.Await` for synchronous-looking code in coroutine context.

```lua
local data, err = ml.Network.RequestAsync('player:getCharacter', {})
if err then
    print('Error:', err)
    return
end
print(data.fullname)
```

### Client Exports

```lua
exports.meta_logic:ServerRequest(route, payload, cb)
exports.meta_logic:ServerRequestAsync(route, payload)
```

## Server: Registering Routes

Register a route handler from any resource:

```lua
exports.meta_logic:RegisterRoute('myresource:doAction', function(source, payload)
    -- Validate
    if not payload.targetId then
        return nil, 'missing targetId'
    end

    -- Process
    local result = doSomething(source, payload.targetId)

    -- Return data (or nil + error string)
    return { success = true, result = result }
end)
```

**Handler signature:** `function(source, payload) -> data, err`

- Return `data` on success
- Return `nil, 'error message'` on failure

## Built-in Routes

meta_logic registers these routes internally:

| Route | Description |
|-------|-------------|
| `player:getSession` | Get the caller's session |
| `player:getCharacter` | Get the caller's character |
| `player:hasPermission` | Check an ACE permission node |
| `player:isAdmin` | Check admin status |
| `player:hasJob` | Check job + optional grade |
| `player:getStateBag` | Get full state bag |
| `vehicle:toggleLock` | Toggle vehicle lock (distance validated) |
| `interaction:checkAccess` | Check ACE permission for interaction |
| `interaction:checkJob` | Check job for interaction |

## Timeout Handling

Requests that don't receive a response within `ml.Config.CacheRequestTimeout` (default 10s) are automatically cleaned up. A background thread sweeps pending requests every 5 seconds.
