# Player

The player module handles the client-side cache of player data and continuous tracking of runtime state.

## Client Cache

When a player loads in, meta_logic server pushes a full sync of their data to the client:

- Account info (source, license, name)
- Character data (name, dob, sex)
- Job and gang
- Permissions
- State bag

After the initial sync, changes are pushed as **patches** — only the changed field is sent.

### Accessing Cached Data

```lua
local data = exports.meta_logic:GetPlayerData()
-- { acct = {...}, character = {...}, job = {...}, gang = {...}, permissions = {...}, state = {...} }

local character = exports.meta_logic:GetCharacter()
-- { firstname = 'John', lastname = 'Doe', fullname = 'John Doe', ... }

local job = exports.meta_logic:GetJob()
-- { name = 'police', label = 'Police', grade = 3, ... }

local gang = exports.meta_logic:GetGang()
-- { name = 'none', label = 'None', grade = 0 }

local loaded = exports.meta_logic:IsPlayerLoaded()
-- true/false
```

All cached data is **read-only** on the client. Changes must go through meta_core on the server.

## Player Tracking

A client-side thread runs every `meta.Config.TrackingInterval` ms (default 200) and updates:

| Key | Description |
|-----|-------------|
| `ped` | Current player ped handle |
| `coords` | Current position (vector3) |
| `heading` | Current heading |
| `isInVehicle` | Whether player is in a vehicle |
| `currentVehicle` | Vehicle entity handle (or nil) |
| `vehicleSeat` | Seat index (or nil) |
| `currentWeapon` | Weapon hash (or nil) |
| `isArmed` | Whether player has a weapon out |
| `isDead` | Whether player ped is dead |

### Accessing Tracked Data

```lua
local coords = exports.meta_logic:GetCoords()
local inVehicle = exports.meta_logic:IsInVehicle()
local vehicle = exports.meta_logic:GetCurrentVehicle()
```

Or through `meta.self` directly:

```lua
meta.self.GetPed()
meta.self.GetCoords()
meta.self.GetHeading()
meta.self.IsInVehicle()
meta.self.GetCurrentVehicle()
meta.self.GetVehicleSeat()
meta.self.GetCurrentWeapon()
meta.self.IsArmed()
meta.self.IsDead()
meta.self.IsHandcuffed()
meta.self.IsLastStand()
meta.self.IsDowned()
```

## Client Events

| Event | Fired When |
|-------|------------|
| `meta_logic:client:playerLoaded` | Full cache sync received |
| `meta_logic:client:playerUnloaded` | Player disconnected |
| `meta_logic:client:jobUpdated` | Job changed |
| `meta_logic:client:gangUpdated` | Gang changed |
| `meta_logic:client:stateChanged` | Server state value changed |

## Server Bridge

On the server side, `meta.Player` provides pass-through functions to meta_core:

```lua
meta.Player.GetAcct(source)
meta.Player.GetChar(source)
meta.Player.HasPermission(source, node)
meta.Player.HasRole(source, role)
meta.Player.HasJob(source, name, minGrade)
meta.Player.HasGang(source, name, minGrade)
meta.Player.IsAdmin(source)
meta.Player.UpdateJob(source, jobData)
meta.Player.UpdateGang(source, gangData)
meta.Player.GetState(source, key)
meta.Player.SetState(source, key, value)
meta.Player.GetStateBag(source)
```

## Resource Restart Handling

When meta_logic restarts, it automatically re-syncs all connected players who already have an active character in meta_core.
